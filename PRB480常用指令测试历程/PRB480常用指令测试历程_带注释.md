# PRB480常用指令测试历程：带注释讲解

> 说明：本说明结合 `../PRB480数据手册V0.0.pdf`。该 PDF 的元数据标题为 `DS28E01-100 DS`，其命令体系与 Maxim/ADI DS28E01-100：1Kb Protected 1-Wire EEPROM with SHA-1 Engine 对应。  
> 下面“手册位置”优先写本地 PDF 中可直接查找的章节/图名/命令名；由于该 PDF 文本流不易直接抽取，建议在 PDF 阅读器中搜索英文命令名，如 `Read ROM`、`Read Memory`、`Write Scratchpad`、`Load First Secret`。

## 0. 先理解这颗芯片在干什么

PRB480/DS28E01-100 是一颗带 SHA-1 引擎的 1-Wire EEPROM/认证芯片。它大致包含这些区域：

- 64-bit ROM ID：芯片唯一身份号，包含 family code、48-bit 序列号、CRC。
- 64-bit Scratchpad：8 字节暂存区，所有写入都先写到这里，再复制到 EEPROM/密钥区。
- EEPROM 数据区：1024 bit，即 4 页，每页 32 字节。
- Secret 密钥区：8 字节，通常不可直接读出，用于认证和 MAC 计算。
- Register/配置区：用于写保护、EPROM 模式、密钥保护等配置。
- SHA-1 引擎：根据 secret、页面数据、ROM ID、challenge/附加数据等生成 160-bit MAC。

手册位置：

- `General Description / Features`：说明 1024-bit EEPROM、64-bit scratchpad、SHA-1、160-bit MAC、64-bit ROM。
- `Overview` / `Figure 1. Block Diagram`：芯片内部模块。
- `Memory Access`：数据页、secret、scratchpad、register page。
- `Figure 2. Hierarchic Structure for 1-Wire Protocol`：1-Wire 访问层级。

基本访问层级是：

```text
1-Wire Reset
  -> ROM Function Command
  -> Memory/SHA Function Command
  -> 数据输入/输出、CRC、等待编程或 SHA 计算、应答
```

注意：`数据读写顺序.docx` 中写了“所有数据读写都是低位在前，高位在后”。这对应 1-Wire 的 LSB first 传输习惯。也就是说发送 `0x33` 时，线上实际 bit 顺序是 bit0 到 bit7。

## 1. 第一步 Read ROM

原始文件：`第一步Read_ROM`

对应手册位置：

- `ROM Function Commands`
- `Read ROM [33h]`
- `64-Bit Lasered ROM` / `Figure 3. 64-Bit Lasered ROM`
- `1-Wire Bus System` / reset + presence pulse

带注释代码：

```c
/* 上电复位 / 1-Wire Reset
 *
 * 作用：
 * 主机先拉低 1-Wire 总线产生 reset pulse，随后释放总线；
 * 芯片如果在线，会返回 presence pulse。
 *
 * 手册位置：
 * - 1-Wire Bus System
 * - Transaction Sequence
 * - Reset pulse / Presence pulse timing
 */
Reset;

/* 等待 1ms，期间保持供电
 *
 * 作用：
 * 给芯片完成复位响应和内部状态稳定的时间。
 * 1-Wire 总线器件在复位后才能接收 ROM 命令。
 *
 * 手册位置：
 * - 1-Wire Signaling
 * - Initialization Procedure
 */
wait_1ms;

/* 发送 Read ROM 命令码 0x33
 *
 * 作用：
 * 当总线上只有一个从设备时，可以用 Read ROM 直接读取该芯片的 64-bit ROM ID。
 * 如果同一条 1-Wire 总线上挂了多个设备，不能直接用 Read ROM，否则多个设备同时返回会冲突。
 *
 * 手册位置：
 * - ROM Function Commands
 * - Read ROM [33h]
 */
Send(8'h33);

/* 读取 8 字节 ROM ID
 *
 * 返回格式通常为：
 * byte0：Family Code
 * byte1~byte6：48-bit 唯一序列号
 * byte7：前 7 字节的 1-Wire CRC
 *
 * 本例读出：
 * af f0 10 04 a0 31 03 c7
 *
 * 手册位置：
 * - 64-Bit Lasered ROM
 * - Figure 3. 64-Bit Lasered ROM
 */
repeat(8) Receive_Byte;
```

这一段的目的：确认芯片在线，并拿到它的唯一身份号。后续 `Match ROM` 会用这个 ID 精确选中这颗芯片。

## 2. 第二步 Read Memory

原始文件：`第二步Read_memory`

对应手册位置：

- `ROM Function Commands`
- `Skip ROM [CCh]`
- `Memory/SHA Function Commands`
- `Read Memory [F0h]`
- `Memory Access`

带注释代码：

```c
/* 上电复位，开始一次新的 1-Wire 事务 */
Reset;

/* 等待芯片复位完成 */
wait_1ms;

/* 发送 Skip ROM 命令 0xCC
 *
 * 作用：
 * 跳过 ROM 匹配，直接让总线上的设备进入后续 Memory/SHA 命令阶段。
 *
 * 使用条件：
 * 总线上只有一颗 PRB480/DS28E01 设备时可以这样做。
 * 如果总线上有多颗 1-Wire 设备，应该用 Match ROM [55h] + 8 字节 ROM ID。
 *
 * 手册位置：
 * - ROM Function Commands
 * - Skip ROM [CCh]
 */
Send(8'hCC);

/* 发送 Read Memory 命令 0xF0
 *
 * 作用：
 * 从指定地址开始顺序读取存储器内容。
 * 该命令通常可以读 EEPROM 数据页和寄存器页，但 secret 密钥区不可直接读出。
 *
 * 手册位置：
 * - Memory/SHA Function Commands
 * - Read Memory [F0h]
 */
Send(8'hF0);

/* 发送起始地址 0x0000
 *
 * 这里分两字节发送：
 * TA1 = 0x00，地址低字节
 * TA2 = 0x00，地址高字节
 *
 * 注意：
 * 1-Wire 字节内部 LSB first；
 * 多字节地址这里按 TA1 后 TA2，也就是低地址字节先发。
 *
 * 手册位置：
 * - Address Registers and Transfer Status
 * - TA1 / TA2 target address
 */
Send(8'h00);
Send(8'h00);

/* 从 0x0000 开始连续读取 160 字节
 *
 * 160 字节 = 0xA0 字节，地址范围大致是 0x0000 ~ 0x009F。
 * DS28E01 的用户 EEPROM 是 4 页 * 32 字节 = 128 字节；
 * 后面还能看到寄存器/状态/ROM ID 映射相关内容。
 *
 * 手册位置：
 * - Memory Access
 * - Memory Map / Register Page
 * - Read Memory [F0h]
 */
repeat(160) Receive_Byte;
```

读出数据的理解：

```text
0x0000~0x007F：主要是 4 页用户 EEPROM，每页 32 字节
0x0080 附近：测试中作为 first secret 写入地址使用
后续区域：配置/寄存器/状态/ROM ID 映射信息，具体以手册 memory map 为准
```

原始记录中后面出现了：

```text
af f0 10 04 a0 31 03 c7
```

这和第一步读到的 ROM ID 一致，说明读出的存储映射里包含了 ROM 相关信息或测试记录把 ROM ID 放在该区域展示。

## 3. 第三步 Load First Secret

原始文件：`第三步load_first_secret`

对应手册位置：

- `Match ROM [55h]`
- `Write Scratchpad [0Fh]`
- `Read Scratchpad [AAh]`
- `Load First Secret [5Ah]`
- `Address Registers and Transfer Status`
- `Writing with Verification`

这一步的核心目的：把 8 字节初始密钥写入 secret 区。示例密钥为：

```text
12 34 56 78 ab cd ef 00
```

带注释代码：

```c
/* 复位，开始新事务 */
Reset;
wait_1ms;

/* 发送 Match ROM 命令 0x55
 *
 * 作用：
 * 用 64-bit ROM ID 精确选中某一颗芯片。
 * 后续命令只有被选中的芯片响应。
 *
 * 手册位置：
 * - ROM Function Commands
 * - Match ROM [55h]
 */
Send(8'h55);

/* 发送之前第一步读到的 8 字节 ROM ID */
Send(8'haf);
Send(8'hf0);
Send(8'h10);
Send(8'h04);
Send(8'ha0);
Send(8'h31);
Send(8'h03);
Send(8'hc7);

/* 发送 Write Scratchpad 命令 0x0F
 *
 * 作用：
 * 写操作不是直接写 EEPROM/secret，而是先写入 8 字节 scratchpad。
 * 写完后通常要 Read Scratchpad 读回 TA1/TA2/E/S 和数据，确认暂存内容正确。
 *
 * 手册位置：
 * - Memory/SHA Function Commands
 * - Write Scratchpad [0Fh]
 * - Writing with Verification
 */
Send(8'h0F);

/* 发送目标地址 0x0080
 *
 * TA1 = 0x80
 * TA2 = 0x00
 *
 * 在这份测试记录中，0x0080 被用作写 first secret 的目标地址。
 * 具体地址含义对应手册 memory map / secret/register 区说明。
 *
 * 手册位置：
 * - Memory Access
 * - Address Registers and Transfer Status
 */
Send(8'h80);
Send(8'h00);

/* 写入 8 字节密钥到 scratchpad */
Send(8'h12);
Send(8'h34);
Send(8'h56);
Send(8'h78);
Send(8'hab);
Send(8'hcd);
Send(8'hef);
Send(8'h00);

/* 接收 CRC-16 校验值
 *
 * 作用：
 * 芯片对 Write Scratchpad 命令、地址、数据计算 CRC16；
 * 主机可用它确认传输没有错误。
 *
 * 手册位置：
 * - Write Scratchpad [0Fh]
 * - CRC-16 Generator / Data Integrity
 */
repeat(2) Receive_Byte;

/* 发送复位指令码：先发 dummy，再发 C0 DE
 *
 * 这不是标准 1-Wire ROM 命令本身，更像测试平台/仿真脚本中的“软复位/事务切换”写法。
 * 目的应是结束当前命令流，让芯片和测试环境进入下一段 Resume/Read Scratchpad。
 *
 * 手册相关：
 * - 1-Wire Transaction Sequence
 * - Reset / Resume Communication
 */
Send(8'hFF);
Send(8'hC0);
Send(8'hDE);

/* 发送 Resume 命令 0xA5
 *
 * 作用：
 * 在前面已经 Match ROM 选中过芯片后，可以用 Resume 跳过再次发送 8 字节 ROM ID。
 *
 * 注意：
 * 这里 0xA5 在 ROM 命令中可表示 Resume；
 * 在 Memory/SHA 命令中，0xA5 又可表示 Read Authenticated Page。
 * 同一个数值处于不同命令层级时含义不同。
 *
 * 手册位置：
 * - ROM Function Commands
 * - Resume [A5h]
 */
Send(8'hA5);

/* 发送 Read Scratchpad 命令 0xAA
 *
 * 作用：
 * 读回 scratchpad 地址寄存器和 8 字节暂存数据。
 * 后续 Load First Secret 需要 TA1、TA2、E/S 作为授权模式。
 *
 * 手册位置：
 * - Memory/SHA Function Commands
 * - Read Scratchpad [AAh]
 * - Address Registers and Transfer Status
 */
Send(8'hAA);

/* 读 3 字节地址/状态：
 * TA1, TA2, E/S
 *
 * E/S 包含：
 * - ending offset
 * - PF：partial byte flag
 * - AA：authorization accepted flag
 *
 * 手册位置：
 * - Address Registers and Transfer Status
 * - Figure 7. Address Registers
 */
repeat(3) Receive_Byte;

/* 读回 8 字节 scratchpad 数据，应该等于刚写入的密钥 */
repeat(8) Receive_Byte;

/* 读 CRC-16 */
repeat(2) Receive_Byte;

/* 再次进入下一段命令 */
Send(8'hC0);
Send(8'hDE);

/* Resume：继续选中前面 Match ROM 的芯片 */
Send(8'hA5);

/* 发送 Load First Secret 命令 0x5A
 *
 * 作用：
 * 把 scratchpad 中的 8 字节内容加载为 first secret。
 * 这是初始化密钥的特殊命令；正常 Copy Scratchpad 写数据一般需要 MAC 授权，
 * 但 Load First Secret 是安装初始 secret 的路径。
 *
 * 手册位置：
 * - Memory/SHA Function Commands
 * - Load First Secret [5Ah]
 */
Send(8'h5A);

/* 发送授权模式中的 TA1/TA2 */
Send(8'h80);
Send(8'h00);

/* 发送 E/S 字节
 *
 * E/S 应来自刚刚 Read Scratchpad 读回的值。
 * 这里写死为 0x5F，说明测试中已知该写入对应的 E/S。
 *
 * 手册位置：
 * - Load First Secret [5Ah]
 * - Address Registers and Transfer Status
 */
Send(8'h5F);

/* 等待内部 EEPROM/secret 写入完成
 *
 * 期间必须保持 1-Wire 供电/总线电压，不能掉电。
 *
 * 手册位置：
 * - Load First Secret [5Ah]
 * - tPROG programming time
 */
wait_programming;

/* 读取 AA 循环应答
 *
 * AAh：写入成功
 * FFh：通常表示失败或未授权
 *
 * 手册位置：
 * - Load First Secret [5Ah]
 */
repeat(2) Receive_Byte;
```

关键点：

- 写 secret 前必须先写 scratchpad。
- `Read Scratchpad` 不是多余步骤，它用于拿到 TA1/TA2/E/S 授权信息。
- `Load First Secret` 依赖这个授权信息。
- 成功标志是 `AA` 循环应答。

## 4. 第四步 Read Authenticated Page

原始文件：`第四步read_auth_page`

对应手册位置：

- `Skip ROM [CCh]`
- `Read Authenticated Page [A5h]`
- `SHA-1 Computation Algorithm`
- `160-bit MAC`
- `Memory/SHA Function Commands`

这一步的核心目的：读取某一页数据，同时让芯片内部 SHA-1 引擎基于 secret、页面数据、ROM ID、附加数据等计算 160-bit MAC，用来验证认证算法是否正确。

带注释代码：

```c
/* 复位，开始新事务 */
Reset;
wait_1ms;

/* 总线上只有一颗芯片，所以跳过 ROM 匹配 */
Send(8'hCC);

/* 发送 Read Authentication Page 命令 0xA5
 *
 * 注意：
 * 0xA5 在 ROM 层是 Resume；
 * 在 Memory/SHA 命令层是 Read Authenticated Page。
 * 这里前面已经发了 Skip ROM，所以此处 0xA5 属于 Memory/SHA 命令。
 *
 * 手册位置：
 * - Memory/SHA Function Commands
 * - Read Authenticated Page [A5h]
 */
Send(8'hA5);

/* 发送读取地址 0x0060
 *
 * TA1 = 0x60
 * TA2 = 0x00
 *
 * 0x0060 对应用户 EEPROM 第 3 页起始地址：
 * page 0: 0x0000~0x001F
 * page 1: 0x0020~0x003F
 * page 2: 0x0040~0x005F
 * page 3: 0x0060~0x007F
 *
 * 手册位置：
 * - Memory Access
 * - Data memory organized as four 32-byte pages
 * - Read Authenticated Page [A5h]
 */
Send(8'h60);
Send(8'h00);

/* 读取 32 字节页面数据 */
repeat(32) Receive_Byte;

/* 读取 FF 字节
 *
 * Read Authenticated Page 的数据阶段之后会有固定填充/分隔字节。
 *
 * 手册位置：
 * - Read Authenticated Page [A5h] command flow
 */
Receive_Byte;

/* 读取 CRC-16 */
repeat(2) Receive_Byte;

/* 等待芯片内部 SHA-1 引擎计算 MAC */
waitCSHA;

/* 读取 160-bit MAC，即 20 字节 */
repeat(20) Receive_Byte;

/* 读取 MAC 数据段后的 CRC-16 */
repeat(2) Receive_Byte;

/* 读取 01 循环/状态响应 */
repeat(2) Receive_Byte;
```

原始记录给出了算法输入：

```text
12 34 56 78 8c 03 14 bc
99 32 af 0b 02 17 54 8c
d9 84 18 20 cb 6e fe 30
ef 70 3b 3a af e1 21 e1
5a ef 39 d0 ff ff ff ff
43 af f0 10 04 a0 31 03
ab cd ef 00 ff ff ff 80
00 00 00 00 00 00 01 b8
```

可以这样理解这 64 字节 SHA-1 输入块：

- 一部分来自 secret：前面写入的 `12 34 56 78 ab cd ef 00` 会被拆入 SHA-1 消息块指定位置。
- 一部分来自目标页数据：这里读的是 `0x0060` 开始的 32 字节。
- 一部分来自 ROM ID：`af f0 10 04 a0 31 03 c7` 中的若干字节参与计算。
- 一部分是命令、地址、填充常量、长度字段等。

输出 MAC：

```text
31 bb 2b 6d
bb 87 d1 c1
64 1a 75 0b
9b 7b 09 31
66 4f 21 ed
```

这就是 20 字节 / 160-bit SHA-1 MAC。主机端如果用同样 secret 和同样输入块计算，结果应该完全一致。

## 5. 第五步 Copy Scratchpad

原始文件：`第五步copy_scratchpad`

对应手册位置：

- `Match ROM [55h]`
- `Write Scratchpad [0Fh]`
- `Read Scratchpad [AAh]`
- `Copy Scratchpad [55h]`
- `SHA-1 Computation Algorithm`
- `Writing with Verification`
- `Write access requires knowledge of the secret and transmitting a 160-bit MAC`

这一步的核心目的：向用户 EEPROM 写入 8 字节数据。和 Load First Secret 不同，普通写入需要主机发送正确的 160-bit MAC 作为授权。

带注释代码：

```c
/* 复位并等待 */
Reset;
wait_1ms;

/* Match ROM：选中指定芯片 */
Send(8'h55);

/* 发送 ROM ID */
Send(8'haf);
Send(8'hf0);
Send(8'h10);
Send(8'h04);
Send(8'ha0);
Send(8'h31);
Send(8'h03);
Send(8'hc7);

/* Write Scratchpad：先把待写入数据放进 scratchpad */
Send(8'h0F);

/* 目标地址 0x0000，即用户 EEPROM page 0 起始位置 */
Send(8'h00);
Send(8'h00);

/* 写入 8 字节用户数据到 scratchpad */
Send(8'h18);
Send(8'h29);
Send(8'h3f);
Send(8'h4e);
Send(8'h5d);
Send(8'h6c);
Send(8'h7b);
Send(8'h8a);

/* 接收 Write Scratchpad 的 CRC-16 */
repeat(2) Receive_Byte;

/* 测试平台/事务切换用复位序列 */
Send(8'hFF);
Send(8'hC0);
Send(8'hDE);

/* Resume：继续操作已选中的芯片 */
Send(8'hA5);

/* Read Scratchpad：读回 TA1、TA2、E/S 和 8 字节数据 */
Send(8'hAA);

/* 读 TA1/TA2/E/S */
repeat(3) Receive_Byte;

/* 读 8 字节 scratchpad 数据 */
repeat(8) Receive_Byte;

/* 读 CRC-16 */
repeat(2) Receive_Byte;
```

到这里为止，只是把数据写进了 scratchpad，还没有真正写入 EEPROM。

继续：

```c
/* 进入下一段命令 */
Send(8'hC0);
Send(8'hDE);

/* Resume：继续操作前面 Match ROM 选中的芯片 */
Send(8'hA5);

/* Copy Scratchpad 命令 0x55
 *
 * 注意：
 * 0x55 在 ROM 层是 Match ROM；
 * 在 Memory/SHA 层是 Copy Scratchpad。
 * 这里前面已经 Resume，所以此处 0x55 是 Memory/SHA 命令。
 *
 * 作用：
 * 将 scratchpad 内容复制到 EEPROM/寄存器目标地址。
 * 对普通写入，芯片要求主机提供正确的 160-bit MAC。
 *
 * 手册位置：
 * - Memory/SHA Function Commands
 * - Copy Scratchpad [55h]
 * - Writing with Verification
 */
Send(8'h55);

/* 发送授权地址 TA1/TA2 */
Send(8'h00);
Send(8'h00);

/* 发送 E/S 字节
 *
 * 应该来自前一个 Read Scratchpad 读回的 E/S。
 * 这里为 0x5F。
 *
 * 手册位置：
 * - Address Registers and Transfer Status
 * - Copy Scratchpad [55h]
 */
Send(8'h5f);

/* 等待芯片 SHA-1 引擎准备/计算 */
waitCSHA;

/* 主机计算 160-bit MAC 并发送给芯片
 *
 * 芯片内部也会基于 secret、旧页面数据、新 scratchpad 数据、
 * ROM ID、地址/命令/常量等计算 MAC。
 * 主机发送的 MAC 必须与芯片内部计算结果一致，芯片才允许真正写 EEPROM。
 *
 * 手册位置：
 * - Copy Scratchpad [55h]
 * - SHA-1 Computation Algorithm
 * - Write access requires knowledge of the secret and transmitting a 160-bit MAC
 */
Send(8'h48);
Send(8'h1a);
Send(8'h61);
Send(8'hd6);
Send(8'h22);
Send(8'hc9);
Send(8'h8a);
Send(8'h18);
Send(8'h19);
Send(8'hd8);
Send(8'hf4);
Send(8'h2c);
Send(8'h1a);
Send(8'hd0);
Send(8'h30);
Send(8'h0f);
Send(8'h39);
Send(8'h14);
Send(8'h67);
Send(8'hb2);

/* 等待 EEPROM 编程完成，期间总线供电不能掉 */
waitprog;

/* 读取 AA 应答
 *
 * AAh：Copy Scratchpad 成功，数据已写入目标 EEPROM。
 * FFh：失败，常见原因是 MAC 不匹配、地址/E/S 不匹配、目标区写保护、PF 置位等。
 *
 * 手册位置：
```

## 6. 最容易混淆的地方

### 6.1 同一个命令码在不同层级有不同含义

```text
0x55
  ROM 层：Match ROM
  Memory/SHA 层：Copy Scratchpad

0xA5
  ROM 层：Resume
  Memory/SHA 层：Read Authenticated Page
```

判断依据是：前面刚发的是 ROM 命令阶段，还是已经通过 `Skip ROM` / `Match ROM` / `Resume` 进入 Memory/SHA 命令阶段。

手册位置：

- `Figure 2. Hierarchic Structure for 1-Wire Protocol`
- `ROM Function Commands`
- `Memory/SHA Function Commands`

### 6.2 写 EEPROM 不是直接写

正确流程：

```text
Write Scratchpad
  -> Read Scratchpad 验证 TA1/TA2/E/S 和数据
  -> Copy Scratchpad
  -> 发送 MAC
  -> 等待 tPROG
  -> 读取 AA 成功应答
  -> Read Memory 读回确认
```

手册位置：

- `Writing with Verification`
- `Write Scratchpad [0Fh]`
- `Read Scratchpad [AAh]`
- `Copy Scratchpad [55h]`

### 6.3 Load First Secret 和 Copy Scratchpad 的区别

```text
Load First Secret:
  用于初始化 secret；
  使用 scratchpad + TA1/TA2/E/S 授权；
  不需要主机发送 160-bit MAC。

Copy Scratchpad:
  用于写用户 EEPROM/寄存器等；
  需要主机知道 secret 并发送正确 160-bit MAC；
  MAC 不匹配则不会写入。
```

手册位置：

- `Load First Secret [5Ah]`
- `Copy Scratchpad [55h]`
- `Features` 中 “Write access requires knowledge of the secret and transmitting a 160-bit MAC”

### 6.4 低位在前

字节内部按 LSB first 走 1-Wire 总线。比如发送 `0x33`，不是从 bit7 开始，而是：

```text
0x33 = 0011 0011b
线上顺序：1,1,0,0,1,1,0,0
```

手册位置：

- `1-Wire Signaling`
- 本目录 `数据读写顺序.docx`

## 7. 五个测试文件串起来的完整目的

```text
第一步 Read ROM
  识别芯片，拿 ROM ID

第二步 Read Memory
  读取全片可见存储内容，确认 memory map 和初始数据

第三步 Load First Secret
  写入 first secret：12 34 56 78 ab cd ef 00

第四步 Read Authenticated Page
  让芯片基于 secret 和指定页面计算 MAC，用于验证认证算法

第五步 Copy Scratchpad
  主机计算并发送 MAC，授权芯片把 scratchpad 数据写入 EEPROM，然后读回确认
```

这套流程跑通后，就说明：

- 1-Wire 复位/读写时序基本正确。
- ROM 命令层和 Memory/SHA 命令层切换正确。
- scratchpad 写入和读回正确。
- secret 初始化正确。
- 芯片 SHA-1 MAC 输出和主机算法一致。
- 主机 MAC 授权写 EEPROM 的流程正确。

后续就可以继续做：

- 写用户区数据。
- 写配置区。
- 锁定 secret。
- 设置页面写保护。
- 做烟弹/主机防伪认证和生产烧录流程。
